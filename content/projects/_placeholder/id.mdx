## Pendekatan

Bagian pendekatan teknis mencakup keputusan arsitektur dan alasan di balik pilihan kunci. Mengapa stack ini dibanding alternatif? Apa batasan yang membentuk desain? Kompromi apa yang diterima dan mengapa?

Misalnya: memilih static export daripada deployment server-rendered mengurangi kompleksitas infrastruktur sekaligus memungkinkan halaman yang dimuat cepat melalui CDN. Komprominya adalah fitur yang berat data atau personal membutuhkan fetching sisi klien, yang dapat diterima untuk situs portofolio namun tidak untuk aplikasi transaksional.

### Gambaran arsitektur

Deskripsi singkat komponen utama sistem dan cara mereka terhubung:

- **Frontend**: Next.js App Router dengan static export, di-deploy ke Vercel
- **Konten**: Model konten MDX berbasis file dengan `gray-matter` untuk parsing frontmatter
- **i18n**: Pergantian locale berbasis route melalui `next-intl` — setiap route ada di `/en/` dan `/id/`
- **Styling**: Tailwind CSS v4 dengan design token yang didefinisikan dalam CSS `@theme`
